# PRD · Research Portfolio

## 제품 목표

김민식의 연구 관심, 프로젝트, 논문, 기록을 한글과 영문으로 명료하게 보여주고, 개발 지식이 많지 않아도 한 파일에서 안전하게 갱신할 수 있는 개인 연구 포트폴리오를 만든다.

## 핵심 사용자

- 연구 주제와 배경을 빠르게 파악하려는 연구자·교수·동료
- 프로젝트와 코드 연결을 확인하려는 협업자
- 자신의 활동을 지속적으로 추가·정리하는 사이트 소유자

## 디자인 원칙

- 편집 연구 노트 같은 정보 위계와 넓은 여백
- Pretendard 본문, Satoshi 영문 제목의 역할 분리
- 미네랄 페이퍼·잉크·버밀리언의 제한된 색 체계
- 의미 없는 카드, 둥근 상태 뱃지, 장식용 이모지, 왼쪽 강조선 금지
- 모션은 질문·근거·기록의 관계를 보여주는 하나의 궤도 오브제에 집중

## Phase 0–6 Acceptance Criteria

### Phase 0 · 콘텐츠 기반

- 기존 저장소와 공개 주소가 확인되어 있다.
- 확인되지 않은 학력·소속·논문을 사실처럼 작성하지 않는다.
- 개인 정보는 사용자가 명시적으로 추가하기 전까지 노출하지 않는다.

### Phase 1 · 정보 구조

- Home, Approach, Work, Profile, Contact 흐름이 존재한다.
- 논문 데이터가 없을 때 Publications가 자동으로 숨는다.
- 콘텐츠 편집 지점이 `content.js`로 모여 있다.

### Phase 2 · 디자인 시스템

- 전역 색·글꼴·간격이 CSS 토큰으로 선언되어 있다.
- 모바일 390px부터 데스크톱 1440px까지 가로 스크롤이 없다.
- UI 아이콘은 Lucide만 사용한다.
- 금지된 왼쪽 강조선 및 기본 알약 상태 뱃지가 없다.

### Phase 3 · 상호작용

- 한글/영문 전환이 전체 핵심 콘텐츠와 메타 정보에 반영된다.
- 모바일 메뉴에 올바른 `aria-expanded` 상태가 있다.
- 연구 루프 네 단계를 버튼과 키보드 포커스로 탐색할 수 있다.
- `prefers-reduced-motion`에서 불필요한 움직임이 멈춘다.
- Lottie는 투명 배경, 60fps, 자연스러운 반복으로 렌더링된다.

### Phase 4 · SEO와 구조화 데이터

- title, description, canonical, Open Graph, Twitter 메타가 있다.
- Person JSON-LD가 현재 언어와 링크를 반영한다.
- `sitemap.xml`, `robots.txt`, 웹 앱 매니페스트가 있다.

### Phase 5 · 접근성·보안·성능

- 건너뛰기 링크, 의미 있는 헤딩 순서, 키보드 포커스가 있다.
- 외부 링크는 안전한 `rel` 속성을 사용한다.
- CSP 기준안과 GitHub Pages 보안 헤더 한계가 문서화되어 있다.
- 의미 없는 파티클·스크롤 라이브러리를 제거해 초기 실행 비용을 줄인다.

### Phase 6 · 품질 및 운영

- 콘텐츠·JavaScript·Lottie 정적 검사가 통과한다.
- Playwright 데스크톱·모바일 동작 검사가 통과한다.
- 대표 화면의 시각 회귀 기준 이미지가 저장된다.
- GitHub 프로필 README 초안과 콘텐츠 편집 가이드가 있다.
